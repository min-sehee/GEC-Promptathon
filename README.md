# 최종 프로젝트 제출 리포지토리

## 개요

이 리포지토리는 **Grammar Error Correction Promptathon** 최종 프로젝트를 제출하기 위한 공식 공간입니다.
모든 프로젝트가 원활하게 취합될 수 있도록, 아래의 가이드라인을 반드시 준수해야 합니다.

---

## 프로젝트 시작 절차

모든 팀은 프로젝트 시작 전, 다음 절차에 따라 팀의 작업 디렉토리를 설정해야 합니다.

1.  **팀 디렉토리 생성**: `projects/` 디렉토리 하위에 팀명으로 된 신규 디렉토리를 생성합니다. 디렉토리명에는 공백 대신 하이픈(-) 사용을 원칙으로 합니다.
    * `예시: projects/FC-Prompthon-팀이름/`

2.  **프로젝트 파일 관리**: 모든 소스 코드, README.md 등 프로젝트와 관련된 모든 산출물은 상기 단계에서 생성한 팀 디렉토리 내부에 위치해야 합니다.

3.  **README 작성**: 각 팀의 디렉토리에는 프로젝트를 상세히 설명하는 `README.md` 파일이 반드시 포함되어야 합니다. README 템플릿은 아래 예시를 참고해주세요.

---

## 디렉토리 구조

각 팀은 `projects/` 하위에 자신만의 디렉토리를 가지며, 팀 내부에서는 아래 구조에 따라 프로젝트를 구성해야 합니다.
```
.
│
├── README.md           # 본 파일
│
└── projects/
    │
    └── [FC_Prompthon_팀이름]/           # 각 팀은 자신의 팀명으로 디렉토리를 생성
        │
        ├── README.md     # 팀별 상세 설명 파일
        └── code/
            ├── generate.py   # 교정 문장 생성 스크립트
            └── prompts.py    # 프롬프트 템플릿 (이 파일을 수정하세요!)

```
---

## 팀 디렉토리 내부 README.md 템플릿

각 팀의 디렉토리 내부에 포함될 `README.md` 파일은 아래 템플릿을 따라 작성해주세요.

```markdown
# [팀명]

---

## 팀원 소개

---

## 프롬프트 템플릿 소개 및 활용한 테크닉

* (예시) 프롬프트에 역할(Persona)을 부여하여 전문성을 강화했습니다.
* (예시) Chain-of-Thought 기법을 활용하여 모델의 추론 과정을 유도했습니다.

---

## 제출 방법
프로젝트 제출은 Fork & Pull Request 방식을 따릅니다. 제출에 관한 상세 절차는 운영진이 별도로 안내한 "최종 프로젝트 제출 가이드" 문서를 참고해주세요.
